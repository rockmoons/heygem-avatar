# 批量输入处理

当用户提供多个链接（聊天换行）或文件路径时，按本指引处理。

---

## 场景 A：聊天多行粘贴

用户直接在对话中换行粘贴多个链接：

```
https://www.douyin.com/video/7661571300523085094
https://v.douyin.com/xxxxx1/
https://www.douyin.com/jingxuan?modal_id=7661571300523085095
```

**处理：**
1. 按换行分割，逐行匹配链接或纯数字 ID
2. 用 SKILL.md Step 1 的规则识别每行
3. 收集到列表，去重（同一 `aweme_id` 只查一次）
4. 判断数量 → 决定是否分批

---

## 场景 B：文件输入

用户指定文件路径：
> 「读 C:\links.txt，全部查出来」
> 「帮我处理这个 excel 里的链接」

### .txt 文件
```
用 cat 或 Read 工具读取 → 按行分割 → 逐行匹配链接/ID
跳过空行和无链接的行
```

### .csv 文件
```
用 Python 内置 csv 模块或 pandas 读取 → 遍历所有列 → 匹配链接/ID
```

### .xlsx 文件
```
pip install openpyxl（如未装）
用 Python openpyxl 读取 → 遍历所有 Sheet、所有行、所有列 → 匹配链接/ID
```

### .docx 文件
```
pip install python-docx（如未装）
用 Python python-docx 提取所有段落文本 → 正则匹配链接/ID
```

---

## 链接去重

提取所有链接后：
- 统一提取出 `aweme_id`（15-25 位数字）
- 按 ID 去重
- 输出：「共提取到 N 个不重复的视频链接」

---

## 批量策略

### 如果 ≤ 50 条
直接逐条处理，不提示

### 如果 > 50 条
```
提示：
「共 {N} 条链接，预计约 {估算时间} 分钟，消耗约 {N×积分单价} 积分。
  将分 {批次数} 批处理，每批 50 条。
  确认继续吗？」

估算公式：
- 每条约 1.5 秒（含 API 延迟）
- 积分单价 = 每条 10 积分（按 API 01 计）
```

等用户回复「确认」「继续」「是」后才执行，回复「取消」「算了」则中止。

---

## 执行

```
逐批循环：
  for batch in chunks(links, 50):
      逐条 curl 调用 API
      间隔 {batch.interval_ms}ms
      更新进度：「已完成 {done}/{total}...」
      单条失败：记录错误原因，继续下一条

完成后汇总：
  ✅ 处理完成：成功 {success} 条，失败 {fail} 条
  {如有失败} 失败列表：{失败的链接及原因}

  💡 导出到 Excel？ 提取文案？
```

---

## 中途停止

用户可以随时说「停」「取消」「够了」，当前批次完成后停止，已处理的数据保留。
